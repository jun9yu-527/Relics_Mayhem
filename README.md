# Relics Mayhem

계명대학교 행소박물관을 테마로 한 2D 물리 퍼즐 게임입니다.  
유물을 좌우로 움직인 뒤 떨어뜨리고, 같은 유물끼리 합쳐 더 높은 단계의 유물로 성장시키며 점수를 올리는 방식으로 진행됩니다.

수박게임류 머지 퍼즐의 규칙을 바탕으로, 박물관 유물을 소재로 만든 게임입니다.  
플레이어는 제한된 공간 안에서 유물이 쌓이는 높이를 관리하며, 같은 유물을 합쳐 더 가치 있는 유물을 만들수록 점수가 올라갑니다.

## 주요 특징

- 상평통보부터 나전 베개모까지 10단계 유물 성장 시스템
- 같은 유물 충돌 시 다음 단계 유물로 합쳐지는 머지 시스템
- 다음에 등장할 유물을 보여주는 프리뷰 UI
- 점수 및 상위 3개 최고 점수 저장
- 미믹, 시계, 블럭 등 게임 흐름에 변수를 만드는 특수 아이템
- 튜토리얼, 유물 정보, 설정 패널, 씬 전환 연출
- BGM / SFX 오디오 버스 기반 사운드 설정
- `user://` 저장소를 활용한 최고 점수 및 오디오 설정 영속 저장
- 일시정지, 재시작, 메인 메뉴 이동을 포함한 기본 게임 흐름

## 실행 방법

1. Godot 4.6 이상을 설치합니다.
2. Godot Project Manager에서 이 저장소 폴더를 가져옵니다.
3. `project.godot`을 열고 실행합니다.
4. 메인 메뉴에서 시작 버튼을 눌러 플레이합니다.

> `project.godot`의 `run/main_scene` 설정에 따라 메인 메뉴 씬부터 시작됩니다.

## 조작법

| 입력 | 동작 |
| --- | --- |
| `A` / `←` | 유물 왼쪽 이동 |
| `D` / `→` | 유물 오른쪽 이동 |
| `Space` | 유물 떨어뜨리기 |
| `Esc` | 게임 일시정지 또는 열린 팝업 닫기 |

게임 플레이 중 `Esc`를 누르면 일시정지 패널이 열립니다.  
일시정지 상태에서는 게임 재개, 설정 열기, 메인 메뉴로 이동이 가능합니다.

## 게임 목표

유물을 최대한 오래 쌓아 올리며 높은 점수를 기록하는 것이 목표입니다.  
같은 유물 2개가 만나면 다음 단계 유물로 합쳐지고, 합쳐진 유물에 따라 점수가 추가됩니다.

공간이 좁아질수록 유물이 상단 데드라인에 가까워집니다.  
유물이 게임 오버 라인 위에 정착한 채 일정 시간 유지되면 게임이 종료되므로, 유물을 어디에 떨어뜨릴지 전략적으로 판단해야 합니다.

---

## 게임 시스템

### 유물 머지

같은 단계의 유물 2개가 충돌하면 다음 단계 유물 1개로 합쳐집니다.  
합쳐진 위치는 두 유물의 중간 지점이며, 성장 효과음과 등장 애니메이션이 함께 재생됩니다.

각 유물 스크립트는 다음 단계 씬을 `NEXT_LEVEL` 상수로 가지고 있습니다.  
충돌한 두 유물이 같은 그룹에 속하고 둘 다 `Processed` 상태가 아닐 때, 두 유물은 제거되고 다음 단계 유물이 생성됩니다.

유물 성장 시 `GameManager.merged_relic_changed` 시그널이 발생하며, HUD는 이 시그널을 받아 현재 유물 설명을 갱신합니다.

### 유물 성장 단계

| 단계 | 유물 | 시대 |
| --- | --- | --- |
| 1 | 상평통보 | 조선 |
| 2 | 주먹도끼 | 구석기 |
| 3 | 짐승얼굴무늬 기와 | 삼국 |
| 4 | 벼루 | 통일신라 |
| 5 | 청동거울 | 청동기 |
| 6 | 비파형 동검 | 청동기 |
| 7 | 빗살무늬 토기 | 신석기 |
| 8 | 금동관 | 삼국 |
| 9 | 고려청자 | 고려 |
| 10 | 나전 베개모 | 조선 |

### 특수 아이템

| 아이템 | 효과 |
| --- | --- |
| 미믹 | 충돌한 유물을 다음 단계로 즉시 성장시킵니다. |
| 시계 | 충돌한 유물을 회전 연출과 함께 제거합니다. |
| 블럭 | 점수 100점 이상부터 확률적으로 등장하는 방해 아이템입니다. |

특수 아이템은 일반 유물과 달리 게임판 상황에 변수를 만듭니다.  
미믹은 원하는 유물을 빠르게 성장시키는 보너스 역할을, 시계는 쌓인 유물을 제거해 공간을 확보하는 역할을 합니다.  
블럭은 합쳐지지 않는 장애물에 가깝기 때문에 위치 관리가 중요합니다.

### 다음 유물 결정

게임 시작 시 첫 유물은 상평통보로 생성됩니다.  
이후 플레이어가 현재 유물을 떨어뜨리고 약 0.5초가 지나면, 프리뷰에 표시된 유물이 상단에 새로 생성됩니다.

다음 유물은 일반 유물 및 특수 아이템 후보 중 확률적으로 결정됩니다.

- 미믹은 낮은 확률로 등장합니다.
- 시계는 미믹보다 더 낮은 확률로 등장합니다.
- 블럭은 점수 100점 달성 후부터 등장 가능하며, 일반 유물이 3회 스폰될 때마다 등장 확률이 1%씩 점진적으로 증가합니다. 블럭이 소환되면 확률은 다시 0%로 초기화됩니다.

### 점수

유물이 합쳐지거나 미믹으로 성장하면 점수가 증가합니다.  
점수는 `GameManager.score`에 저장되고, HUD는 매 프레임 현재 점수를 확인해 화면에 반영합니다.

게임오버 시 현재 점수는 최고 점수 목록에 반영됩니다.  
최고 점수는 높은 순으로 정렬되어 상위 3개만 유지됩니다.

### 게임 오버

유물이 상단 게임 오버 라인 위에 정착한 상태로 일정 시간 유지되면 게임 오버가 됩니다.  
경고 영역에 유물이 감지되면 데드라인 표시가 페이드 인됩니다.

순간적으로 튀어오른 유물 때문에 게임 오버가 즉시 발생하지 않도록 설계되어 있습니다.  
유물이 일정 속도 이하로 정착했거나 `sleeping` 상태이며, 게임 오버 라인을 넘은 상태가 일정 시간 지속될 때 최종 게임 오버로 처리됩니다.

게임 오버 시 다음 흐름이 실행됩니다.

1. `GameManager.set_game_over(true)` 호출
2. 현재 점수 기준 최고 점수 갱신
3. HUD 게임오버 패널 표시 및 최종 점수 출력
4. 게임오버 효과음 재생
5. 다시 시작 또는 메인 메뉴 이동 선택 가능

### 씬 전환

메인 메뉴에서 게임을 시작하면 `TransitionLayer`가 다음 순서로 전환 연출을 실행합니다.

1. 검은 화면 페이드 인
2. 유물 10개 원형 배치 애니메이션
3. `Ready` / `Go!` 텍스트 및 효과음 표시
4. 원형 리빌 연출로 게임 화면 드러남

게임 중 메인 메뉴로 돌아갈 때는 페이드 인/아웃 전환을 사용합니다.  
`TransitionLayer`는 Autoload로 등록되어 어떤 씬에서도 호출할 수 있습니다.

### 오디오

오디오는 Godot의 버스 시스템을 사용합니다.

| 버스 | 역할 |
| --- | --- |
| `Master` | 전체 볼륨 |
| `BGM` | 배경음악 볼륨 |
| `SFX` | 효과음 볼륨 |

설정 패널에서 각 버스의 볼륨과 음소거 여부를 조정할 수 있습니다.  
설정값은 `user://settings.cfg`에 저장되며, 다음 실행 시 자동으로 불러옵니다.

## 저장 데이터

게임은 Godot의 사용자 저장 경로인 `user://`를 사용합니다.  
실제 경로는 실행 환경과 운영체제에 따라 달라집니다.

| --- | --- |

| `user://settings.cfg` | Master / BGM / SFX 볼륨 및 음소거 설정 |

## 개발 메모

- `GameManager`와 `TransitionLayer`는 Autoload로 등록되어 어떤 씬에서도 접근할 수 있습니다.
- 각 유물 스크립트는 자신의 다음 단계 씬을 `NEXT_LEVEL`로 보유하며, 머지 및 미믹 시스템에서 활용됩니다.
- 충돌 중복 처리는 `Processed` 그룹으로 방지합니다.
- 떨어진 유물은 `FallenRelics` 그룹에 추가되어 게임오버 판정, 특수 아이템 충돌, 물리 상태 추적에 사용됩니다.
- 조작 중인 유물은 중력값 0으로 좌우 이동하다가, 떨어뜨리는 순간 중력이 활성화됩니다.
- 데드라인과 가이드라인은 상황에 따라 `Line2D` 노드 또는 `_draw()` 기반으로 표시됩니다.
- `.png.import`, `.mp3.import` 등 Godot import 메타 파일은 프로젝트와 함께 유지해야 합니다.
