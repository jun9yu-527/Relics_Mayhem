# Relics Mayhem

한국 유물을 테마로 한 2D 물리 퍼즐 게임입니다. 유물을 좌우로 움직인 뒤 떨어뜨리고, 같은 유물끼리 합쳐 더 높은 단계의 유물로 성장시키며 점수를 올리는 방식으로 진행됩니다.

`Relics Mayhem`은 수박게임류 머지 퍼즐의 규칙을 바탕으로, 한국의 시대별 유물을 성장 단계로 구성한 게임입니다. 플레이어는 제한된 공간 안에서 유물이 쌓이는 높이를 관리해야 하며, 같은 유물을 합쳐 더 가치 있는 유물을 만들수록 점수가 올라갑니다.

## 프로젝트 정보

- 엔진: Godot 4.6
- 장르: 2D 물리 퍼즐 / 머지 게임
- 해상도: 1280 x 720
- 시작 씬: `scenes/main_menu/main_menu.tscn`
- 게임 씬: `scenes/main/main.tscn`
- 프로젝트 설정: `project.godot`
- 게임 버전: 1.0

## 주요 특징

- 상평통보부터 나전베개까지 이어지는 한국 유물 성장 단계
- 같은 유물 충돌 시 다음 단계 유물로 합쳐지는 머지 시스템
- 다음에 등장할 유물을 보여주는 프리뷰 UI
- 점수 및 상위 3개 최고 점수 저장
- 미믹, 시계, 블록 등 특수 아이템
- 튜토리얼, 유물 정보, 설정 패널, 전환 연출
- BGM/SFX 오디오 버스 기반 사운드 설정
- `user://` 저장소를 활용한 최고 점수 및 오디오 설정 저장
- 일시정지, 재시작, 메인 메뉴 이동을 포함한 기본 게임 흐름

## 게임 목표

게임의 목표는 유물을 최대한 오래 쌓아 올리며 높은 점수를 기록하는 것입니다. 같은 유물 2개가 만나면 다음 단계 유물로 합쳐지고, 합쳐진 유물에 따라 점수가 추가됩니다.

공간이 좁아질수록 유물이 상단 데드라인에 가까워집니다. 유물이 게임 오버 라인 위에 정착한 채 일정 시간 유지되면 게임이 종료되므로, 유물을 어디에 떨어뜨릴지 계속 판단해야 합니다.

## 조작법

| 입력 | 동작 |
| --- | --- |
| `A` / `←` | 유물 왼쪽 이동 |
| `D` / `→` | 유물 오른쪽 이동 |
| `Space` | 유물 떨어뜨리기 |
| `Esc` | 게임 일시정지 또는 열린 팝업 닫기 |

게임 플레이 중 `Esc`를 누르면 일시정지 패널이 열립니다. 일시정지 상태에서는 게임을 재개하거나 설정을 열거나 메인 메뉴로 돌아갈 수 있습니다.

## 화면 구성

- 메인 메뉴: 게임 시작, 튜토리얼, 유물 정보, 설정, 종료 버튼을 제공합니다.
- 튜토리얼: 3장의 안내 이미지로 기본 플레이 방법을 설명합니다.
- 정보창: 3장의 안내 이미지로 게임 내 유물 정보를 보여줍니다.
- 게임 화면: 현재 점수, 최고 점수 1~3위, 다음 유물 프리뷰, 현재 유물 설명, 일시정지/게임오버 UI를 표시합니다.
- 설정 패널: Master, BGM, SFX 볼륨과 음소거 상태를 조정합니다.

## 실행 방법

1. Godot 4.6 이상을 설치합니다.
2. Godot Project Manager에서 이 저장소 폴더를 가져옵니다.
3. `project.godot`을 열고 실행합니다.
4. 메인 메뉴에서 시작 버튼을 눌러 플레이합니다.

Godot 에디터에서 직접 실행할 경우 `project.godot`의 `run/main_scene` 설정에 따라 메인 메뉴 씬부터 시작됩니다.

## 빌드 및 내보내기

프로젝트에는 `export_presets.cfg`가 포함되어 있습니다. Godot의 `Project > Export` 메뉴에서 프리셋을 확인한 뒤 필요한 플랫폼으로 내보낼 수 있습니다.

내보내기 전에 다음 항목을 확인하는 것을 권장합니다.

- Godot 4.6 기준으로 프로젝트가 정상 실행되는지 확인
- BGM/SFX 버스가 `Master`, `BGM`, `SFX` 이름으로 유지되는지 확인
- 이미지, 폰트, 사운드 리소스가 누락되지 않았는지 확인
- 메인 메뉴에서 게임 씬으로 전환되는지 확인
- 게임오버 후 재시작과 메뉴 이동이 정상 작동하는지 확인

## 게임 시스템

### 유물 머지

같은 단계의 유물 2개가 충돌하면 다음 단계 유물 1개로 합쳐집니다. 합쳐진 위치는 두 유물의 중간 지점이며, 성장 효과음과 등장 애니메이션이 함께 재생됩니다.

각 유물 스크립트는 다음 단계 씬을 `NEXT_LEVEL` 상수로 가지고 있습니다. 충돌한 두 유물이 같은 그룹에 속하고 둘 다 아직 `Processed` 상태가 아니라면, 두 유물은 제거되고 다음 단계 유물이 생성됩니다.

유물 성장 시 `GameManager.merged_relic_changed` 시그널이 발생하며, HUD는 이 시그널을 받아 현재 유물 설명을 갱신합니다.

### 유물 성장 단계

| 단계 | 유물 | 시대/설명 |
| --- | --- | --- |
| 1 | 상평통보 | 조선 시대의 대표적인 청동 주화 |
| 2 | 주먹도끼 | 구석기 시대의 사냥, 도살, 채집 도구 |
| 3 | 짐승얼굴무늬 기와 | 삼국 시대 지붕 장식 기와 |
| 4 | 벼루 | 통일신라 시대의 다각연 형태 벼루 |
| 5 | 청동거울 | 청동기 시대의 권력과 의례 상징물 |
| 6 | 비파형 동검 | 청동기 시대의 대표적인 동검 |
| 7 | 빗살무늬 토기 | 신석기 시대의 보관 및 조리용 토기 |
| 8 | 금동관 | 삼국 시대의 화려한 고대 왕관 |
| 9 | 고려청자 | 고려 시대의 청록색 도자기 |
| 10 | 나전 베개모 | 조선 시대 나전 기법 장식품 |

### 특수 아이템

- 미믹: 충돌한 유물을 다음 단계로 성장시킵니다.
- 시계: 충돌한 유물을 회전 연출과 함께 제거합니다.
- 블록: 일정 점수 이후 확률적으로 등장하는 방해 아이템입니다.

특수 아이템은 일반 유물과 다르게 게임판 상황에 변수를 만듭니다. 미믹은 원하는 유물을 빠르게 성장시킬 수 있는 보너스 역할을 하고, 시계는 쌓인 유물을 제거해 공간을 확보하는 역할을 합니다. 블록은 병합되지 않는 장애물에 가까우므로 위치 관리가 중요합니다.

### 다음 유물 결정

게임 시작 시 첫 유물은 상평통보로 생성됩니다. 이후에는 다음 유물 프리뷰에 표시된 유물이 스폰되며, 플레이어가 현재 유물을 떨어뜨리고 약 0.5초가 지나면 새 유물이 상단에 생성됩니다.

다음 유물은 일반 유물과 특수 아이템 후보 중에서 확률적으로 결정됩니다.

- 미믹은 낮은 확률로 등장합니다.
- 시계는 더 낮은 확률로 등장합니다.
- 블록은 점수가 일정 기준 이상일 때부터 등장할 수 있습니다.
- 블록 확률은 일정 횟수의 일반 유물 스폰 이후 점진적으로 증가합니다.

### 점수

유물이 합쳐지거나 미믹으로 성장하면 점수가 증가합니다. 점수는 `GameManager.score`에 저장되고, HUD는 매 프레임 현재 점수를 확인해 화면에 반영합니다.

게임오버가 되면 현재 점수는 최고 점수 목록에 반영됩니다. 최고 점수는 높은 순서로 정렬되어 상위 3개만 유지됩니다.

### 게임 오버

유물이 상단 게임 오버 라인 위에 정착한 상태로 일정 시간 유지되면 게임 오버가 됩니다. 경고 영역에 유물이 감지되면 데드라인 표시가 페이드 인됩니다.

게임 오버 판정은 순간적으로 튀어오른 유물 때문에 바로 발생하지 않도록 설계되어 있습니다. 유물이 일정 속도 이하로 정착했거나 sleeping 상태이며, 게임 오버 라인을 넘은 상태가 일정 시간 유지될 때 최종 게임 오버로 처리됩니다.

게임 오버가 되면 다음 흐름이 실행됩니다.

1. `GameManager.set_game_over(true)` 호출
2. 현재 점수 기준 최고 점수 갱신
3. HUD 게임오버 패널 표시
4. 최종 점수 표시
5. 게임오버 효과음 재생
6. 다시 시작 또는 메인 메뉴 이동 선택 가능

### 최고 점수

게임 오버 시 현재 점수를 기준으로 상위 3개의 최고 점수를 갱신합니다. 기록은 Godot의 `user://high_scores.save` 경로에 저장됩니다.

### 오디오 설정

오디오는 Godot의 버스 시스템을 사용합니다.

- `Master`: 전체 볼륨
- `BGM`: 배경음악 볼륨
- `SFX`: 효과음 볼륨

설정 패널에서는 각 버스의 볼륨과 음소거 여부를 조정할 수 있습니다. 설정값은 `user://settings.cfg`에 저장되며, 다음 실행 시 다시 불러옵니다.

### 씬 전환

메인 메뉴에서 게임을 시작하면 `TransitionLayer`가 검은 화면 페이드, 유물 원형 배치 애니메이션, `Ready`/`Go!` 표시, 원형 리빌 연출을 순서대로 실행합니다.

게임 중 메인 메뉴로 돌아갈 때는 페이드 인/아웃 전환을 사용합니다. 전환 레이어는 Autoload로 등록되어 어떤 씬에서도 호출할 수 있습니다.

## 개발 메모

- `GameManager`는 Autoload로 등록되어 점수, 게임 오버 상태, 최고 점수, 머지 효과음을 관리합니다.
- `TransitionLayer`는 Autoload로 등록되어 씬 전환과 설정 패널을 처리합니다.
- 각 유물 스크립트는 자신의 다음 단계 씬을 `NEXT_LEVEL`로 가지고 있어 머지 및 미믹 시스템에서 활용됩니다.
- 유물과 아이템의 충돌 중복 처리는 `Processed` 그룹으로 방지합니다.
- 떨어진 유물은 `FallenRelics` 그룹에 추가되어 게임오버 판정, 특수 아이템 충돌, 물리 상태 추적에 사용됩니다.
- 게임 화면의 조작 가능한 유물은 중력값을 0으로 둔 상태에서 좌우 이동하다가, 떨어뜨리는 순간 중력값을 활성화합니다.
- `SpawnController/Timer`는 유물을 떨어뜨린 뒤 다음 유물이 생성되기까지의 짧은 간격을 담당합니다.
- 데드라인과 가이드라인은 상황에 따라 `Line2D` 노드 또는 `_draw()` 기반 그리기로 표시됩니다.

## Autoload

`project.godot`에는 다음 Autoload가 등록되어 있습니다.

| 이름 | 경로 | 역할 |
| --- | --- | --- |
| `GameManager` | `res://manager/game_manager.gd` | 전역 점수, 게임오버, 최고 점수, 머지 이벤트 관리 |
| `TransitionLayer` | `res://scenes/transition/transition_layer.tscn` | 씬 전환과 설정 패널 관리 |

## 입력 설정

`project.godot`의 입력 맵에는 기본 UI 액션이 게임 조작에 사용됩니다.

| 액션 | 사용 키 | 용도 |
| --- | --- | --- |
| `ui_left` | `A`, 왼쪽 방향키 | 유물 왼쪽 이동 |
| `ui_right` | `D`, 오른쪽 방향키 | 유물 오른쪽 이동 |
| `ui_select` / `ui_accept` | `Space` | 유물 떨어뜨리기 |
| `ui_cancel` | `Esc` | 일시정지, 팝업 닫기 |

## 저장 데이터

게임은 Godot의 사용자 저장 경로인 `user://`를 사용합니다. 이 경로는 실행 환경과 운영체제에 따라 실제 위치가 달라집니다.

| 파일 | 내용 |
| --- | --- |
| `user://high_scores.save` | 상위 3개 최고 점수 |
| `user://settings.cfg` | Master/BGM/SFX 볼륨 및 음소거 설정 |

## 리소스

Godot 프로젝트 특성상 `.png.import`, `.mp3.import`, `.ttf.import` 같은 import 메타 파일도 함께 유지해야 합니다.

## 작업 시 참고

- 새 유물을 추가하려면 유물 이미지, 씬, 충돌 형태, 그룹, 머지 스크립트, 다음 단계 연결을 함께 확인해야 합니다.
- 유물 설명을 추가하거나 수정하려면 `scenes/main/hud.gd`의 `_on_relic_merged()` 내용을 갱신해야 합니다.
- 특수 아이템 등장 확률은 `scenes/main/main.gd`의 `_on_timer_timeout()`에서 조정합니다.
- 최고 점수 초기화가 필요하면 `user://high_scores.save`를 삭제하거나 저장 로직을 임시 수정할 수 있습니다.
- 오디오 설정 초기화가 필요하면 `user://settings.cfg`를 삭제하면 됩니다.
- 새 사운드를 사용할 때는 해당 플레이어의 bus가 `BGM` 또는 `SFX`로 올바르게 지정되어 있는지 확인합니다.

## 알려진 주의사항

- 저장 파일은 로컬 사용자 경로에 생성되므로 저장소에는 포함되지 않습니다.
- Godot import 메타 파일을 삭제하면 에디터에서 리소스를 다시 가져와야 할 수 있습니다.
- 파일명에 괄호가 포함된 유물 리소스가 많으므로 경로를 직접 입력할 때 오타에 주의해야 합니다.
- 프로젝트는 Godot 4.6 기준 설정을 사용하므로, 낮은 버전에서 열 경우 호환성 문제가 생길 수 있습니다.
